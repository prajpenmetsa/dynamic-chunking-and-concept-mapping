# EVALUATION RUBRICS - QUICK REFERENCE GUIDE

## ABCD Framework (1-5 Scale)

### A - AUDIENCE (Who will learn?)
| Score | Criteria |
|-------|----------|
| **5** | Explicitly states learner (e.g., "Students will...", "Learners will...") |
| **4** | Audience implicit but clear from context |
| **3** | Audience can be inferred but not explicit; some ambiguity |
| **2** | Audience unclear or could apply to multiple groups |
| **1** | No identifiable audience; generic statement |

### B - BEHAVIOR (What will they DO?)
| Score | Criteria |
|-------|----------|
| **5** | Precise, observable, measurable action verb (analyze, design, implement, evaluate) |
| **4** | Clear action verb but slightly less measurable (understand, explain, describe) |
| **3** | Vague or weak action verb (know, learn, be aware of) |
| **2** | Passive or non-behavioral verb (be exposed to, appreciate) |
| **1** | No action verb or purely subjective verb |

### C - CONDITION (Under what circumstances?)
| Score | Criteria |
|-------|----------|
| **5** | Explicit conditions stated (e.g., "After completing...", "Using...", "Given...") |
| **4** | Conditions implied by content but not stated; context clear |
| **3** | Some contextual hints but conditions vague |
| **2** | Missing meaningful conditions; no context |
| **1** | Completely absent; no indication of circumstances |

### D - DEGREE (How well?)
| Score | Criteria |
|-------|----------|
| **5** | Specific performance standard (e.g., "with 90% accuracy", "at least 3 differences") |
| **4** | Implicit standard that's reasonably clear (e.g., "effectively", "accurately") |
| **3** | Vague standard (e.g., "well", "properly"); unclear success criteria |
| **2** | Minimal or weak standard; ambiguous |
| **1** | No standard specified; impossible to determine achievement |

## SMART Framework (1-5 Scale)

### S - SPECIFIC (Is it focused and clear?)
| Score | Criteria |
|-------|----------|
| **5** | Highly specific with exact concepts/skills named (e.g., "Implement Banker's algorithm") |
| **4** | Specific but broader area (e.g., "Analyze deadlock prevention strategies") |
| **3** | Somewhat specific but includes vague terms (e.g., "Understand OS concepts") |
| **2** | Very broad or generic (e.g., "Learn about OS") |
| **1** | Completely vague or meaningless (e.g., "Gain knowledge") |

### M - MEASURABLE (Can achievement be assessed?)
| Score | Criteria |
|-------|----------|
| **5** | Explicit success criteria or metrics (e.g., "correctly implement 3 algorithms") |
| **4** | Uses measurable verbs (analyze, design, implement); clear assessment path |
| **3** | Can be measured but method unclear; weak measurable verb (explain, describe) |
| **2** | Difficult to measure objectively; weak verbs (understand, know) |
| **1** | Cannot be measured meaningfully; purely subjective |

### A - ACHIEVABLE (Is it realistic for target learners?)
| Score | Criteria |
|-------|----------|
| **5** | Perfectly scoped for course level; neither trivial nor impossible |
| **4** | Realistic but might be slightly challenging/easy; generally appropriate |
| **3** | Questionable difficulty level; might be too advanced or too basic |
| **2** | Likely unrealistic; too complex or too simple for context |
| **1** | Clearly impossible or insulting; requires unavailable prerequisites or trivial |

### R - RELEVANT (Does it align with course goals?)
| Score | Criteria |
|-------|----------|
| **5** | Directly tied to core course concepts; essential learning |
| **4** | Relevant to course domain; supports main goals |
| **3** | Tangentially related; weak connection to course |
| **2** | Questionable relevance; peripheral topic |
| **1** | Completely irrelevant; doesn't belong in this course |

### T - TIME-BOUND (Is timeframe clear?)
| Score | Criteria |
|-------|----------|
| **5** | Explicit timeframe (e.g., "by end of course", "after module 3") |
| **4** | Timeframe implicit but reasonable (assumes end of course) |
| **3** | Timeframe vague; multiple interpretations |
| **2** | No clear timeframe; unclear when expected |
| **1** | Timeframe absent or makes no sense |

## Bloom's Taxonomy (1-5 Scale)

### OVERALL TAXONOMY ALIGNMENT
| Score | Criteria |
|-------|----------|
| **5** | Precise Bloom's verb from correct level; perfect cognitive demand match |
| **4** | Appropriate Bloom's verb; minor mismatch in complexity |
| **3** | Bloom's verb present but complexity doesn't match level |
| **2** | Weak or incorrect verb for intended level; unclear cognitive demand |
| **1** | No Bloom's verb or completely wrong level |

### VERB ACCURACY (Does verb match cognitive level?)
| Score | Criteria |
|-------|----------|
| **5** | Perfect verb for stated level (e.g., "analyze" for Analyze level) |
| **4** | Appropriate verb with minor issues |
| **3** | Approximate match; verb from adjacent level |
| **2** | Wrong verb for level; significant mismatch |
| **1** | No Bloom's verb or completely inappropriate |

### COGNITIVE DEMAND (Does task match verb complexity?)
| Score | Criteria |
|-------|----------|
| **5** | Task complexity perfectly matches verb level |
| **4** | Good match with minor discrepancies |
| **3** | Partial match; task is easier/harder than verb suggests |
| **2** | Significant mismatch between task and verb |
| **1** | Complete mismatch; task doesn't require stated cognitive level |

### LEVEL CLASSIFICATION (Can you classify unambiguously?)
| Score | Criteria |
|-------|----------|
| **5** | Clearly belongs to single Bloom's level; no ambiguity |
| **4** | Mostly clear with minor edge case considerations |
| **3** | Ambiguous; could be 2 different levels |
| **2** | Very unclear; spans multiple levels |
| **1** | Impossible to classify into Bloom's taxonomy |

### PROGRESSION (Do LOs build from lower to higher levels?)
| Score | Criteria |
|-------|----------|
| **5** | Clear progression from Remember → Create; logical building |
| **4** | Mostly progressive; minor sequencing issues |
| **3** | Some progression but gaps or illogical jumps |
| **2** | Little progression; mostly random ordering |
| **1** | No progression or reversed (high to low) |

## Bloom's Level-Specific Verbs

### Remember (Recall facts, terms, basic concepts)
**Appropriate verbs**: Define, List, Recall, Identify, Label, Name, State, Recognize
- **5**: Pure recall, no interpretation required
- **3**: Mix of recall and understanding
- **1**: Requires higher-order thinking

### Understand (Explain ideas, concepts)
**Appropriate verbs**: Explain, Describe, Summarize, Paraphrase, Interpret, Classify, Compare
- **5**: Requires explanation in own words or interpretation
- **3**: Borders on recall or application
- **1**: Pure recall or analysis level

### Apply (Use knowledge in new situations)
**Appropriate verbs**: Apply, Implement, Execute, Use, Demonstrate, Solve, Compute
- **5**: Clear application to new scenario; execution focus
- **3**: Borders on understanding or analysis
- **1**: No clear application component

### Analyze (Break down, examine relationships)
**Appropriate verbs**: Analyze, Examine, Compare, Contrast, Differentiate, Distinguish, Investigate
- **5**: Breaks down into components, finds relationships, examines structure
- **3**: More like application or evaluation
- **1**: Lower-level task mislabeled

### Evaluate (Judge, critique, assess)
**Appropriate verbs**: Evaluate, Assess, Critique, Judge, Justify, Argue, Defend
- **5**: Makes judgments, defends positions, critiques using standards
- **3**: Borders on analysis or creation
- **1**: No evaluative judgment required

### Create (Design, construct, develop new things)
**Appropriate verbs**: Design, Develop, Create, Construct, Formulate, Plan, Compose
- **5**: Produces something original/new; synthesizes multiple concepts
- **3**: More like apply or evaluate; not truly creating
- **1**: Lower-level task mislabeled

## Consistency Interpretation

### Standard Deviation Thresholds
| StdDev | Interpretation | Symbol |
|--------|----------------|--------|
| ≤ 0.5  | Consistent evaluation | ✓ |
| 0.5-1.0 | Moderately variable | ⚠️ |
| > 1.0  | Highly inconsistent | ❌ |

### Composite Score Interpretation
| Score Range | Quality Level | Action |
|-------------|--------------|--------|
| 4.0 - 5.0 | Excellent | Minor refinements only |
| 3.0 - 3.9 | Good | Targeted improvements |
| 2.0 - 2.9 | Adequate | Significant revision needed |
| 1.0 - 1.9 | Poor | Major restructuring required |
| < 1.0 | Unacceptable | Complete rewrite |

## Critical Evaluation Questions Checklist

### ABCD Framework
- [ ] Can you identify WHO is learning without ambiguity?
- [ ] Is the behavior OBSERVABLE and MEASURABLE?
- [ ] Are CONDITIONS/CONTEXT stated or strongly implied?
- [ ] Is there a STANDARD defining success?
- [ ] Could this be assessed objectively in an exam?

### SMART Framework
- [ ] Would a student know EXACTLY what to learn?
- [ ] Can you create an objective assessment?
- [ ] Is it achievable within one semester?
- [ ] Does it belong in THIS course?
- [ ] Is there a clear timeframe?

### Bloom's Taxonomy
- [ ] Does the verb match the actual cognitive demand?
- [ ] Can you classify into a SINGLE level?
- [ ] Does task complexity match the verb?
- [ ] Do LOs progress from foundational to advanced?
- [ ] Are lower levels prerequisites for higher?

## Quick Decision Tree

### "Is my LO good?"

```
Start
  ↓
Does it use an ACTION VERB? 
  NO → Score ≤ 2 (Fix behavior component)
  YES ↓
  
Is the verb from Bloom's Taxonomy?
  NO → Consider revision (Better alignment)
  YES ↓
  
Can you measure success objectively?
  NO → Score ≤ 3 (Add measurable criteria)
  YES ↓
  
Are audience, conditions, and degree clear?
  NO → Score 3-4 (Add missing components)
  YES ↓
  
Is it specific to course content?
  NO → Score ≤ 3 (Make more specific)
  YES ↓
  
Is it achievable in timeframe?
  NO → Rescope or split
  YES ↓
  
✓ Score likely 4-5 (High quality LO)
```

## Example Annotations

### ❌ Poor LO (Score: 2.1)
**"Students will learn about operating systems."**

| Criterion | Score | Issue |
|-----------|-------|-------|
| Audience | 5 | ✓ Clear ("Students") |
| Behavior | 1 | ❌ "Learn" is not observable |
| Condition | 1 | ❌ No context specified |
| Degree | 1 | ❌ No standard defined |
| Specific | 1 | ❌ Too vague ("about OS") |
| Measurable | 1 | ❌ Cannot assess objectively |
| Bloom's | 1 | ❌ No Bloom's verb |

### ⚠️ Adequate LO (Score: 3.2)
**"Explain the concepts of process synchronization."**

| Criterion | Score | Issue |
|-----------|-------|-------|
| Audience | 3 | ⚠️ Implicit audience |
| Behavior | 4 | ✓ "Explain" is measurable |
| Condition | 3 | ⚠️ Context unclear |
| Degree | 2 | ❌ No standard |
| Specific | 4 | ✓ Names specific topic |
| Measurable | 4 | ✓ Can test via exam |
| Bloom's | 4 | ✓ "Explain" = Understand level |

### ✓ Strong LO (Score: 4.4)
**"Students will analyze at least three deadlock prevention algorithms by comparing their trade-offs in terms of resource utilization and performance overhead."**

| Criterion | Score | Strength |
|-----------|-------|----------|
| Audience | 5 | ✓ "Students" explicit |
| Behavior | 5 | ✓ "Analyze" and "comparing" |
| Condition | 4 | ✓ Context: deadlock algorithms |
| Degree | 5 | ✓ "At least three", specific criteria |
| Specific | 5 | ✓ Names exact algorithms and criteria |
| Measurable | 5 | ✓ Can grade analysis quality |
| Bloom's | 5 | ✓ "Analyze" = Analyze level, task matches |
